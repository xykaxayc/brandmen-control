# Brandmen deployment protocol v2

Статус: переходная реализация в ветке `codex/deployment-v2`.

Legacy HTTP/ADB остаётся доступным для старых плееров. Новый режим включается,
только если планшет сообщает `protocol_version: 2` и был повторно сопряжён с
пультом для передачи индивидуального API-токена.

## Идентичность

- `device_id` строится из `ANDROID_ID` и не зависит от IP.
- IP обновляется при повторной регистрации того же `device_id`.
- Каждый планшет создаёт случайный 256-битный API-токен.
- V2 API требует `Authorization: Bearer <token>`.
- Токен передаётся пульту только в коротком режиме локального сопряжения.

## Manifest

`deployment_id` — SHA-256 канонического JSON без `created_at`. Он зависит от:

- версии протокола;
- версии профиля конвертации;
- логических имён, размеров и SHA-256 файлов;
- порядка плейлиста;
- `playlist_hash`.

Одинаковый набор всегда получает одинаковый ID. Изменение файла при сохранении
имени и размера создаёт новый ID.

## API планшета

### `GET /api/v2/capabilities`

Возвращает возможности, `device_id` и признак обязательной авторизации.

### `POST /api/v2/prepare`

Принимает manifest, проверяет его структуру, `playlist_hash`,
`deployment_id` и свободное место. Сохраняет manifest в staging и возвращает:

- отсутствующие SHA-256;
- размеры сохранённых `.part`;
- готовность к commit.

### `POST /api/v2/blob/<sha256>?offset=<n>&total=<n>`

Продолжает blob с подтверждённого смещения. Перед публикацией проверяет:

- принятый `Content-Length`;
- полный размер;
- SHA-256.

Готовый blob публикуется только после всех проверок.

### `POST /api/v2/commit`

Проверяет наличие всех blobs, сохраняет manifest и атомарно заменяет
`active.json`. Предыдущий ID сохраняется в `previous.json`. После commit
плеер перечитывает активный manifest.

### `POST /api/v2/rollback`

Меняет местами active и previous, затем перезапускает воспроизведение.

### `GET /api/v2/status?deployment_id=<id>`

Возвращает active/previous, playlist hash, отсутствующие blobs и `.part`.

## Проверка применения

Пульт не ограничивается `playing=true`. Он проверяет:

- точный `activeDeploymentId`;
- точный `playlistHash`;
- непустой `currentFileSha256`;
- `playing=true`;
- изменение `positionMs` между двумя опросами.

Если проверка не проходит, пульт запрашивает rollback и запускает предыдущую
версию.

## Desired state

Пульт хранит `desiredDeploymentId` для каждого устройства. Статусы обновляются
раз в минуту. Если V2-планшет вернулся в сеть и его active ID отличается,
пульт автоматически применяет сохранённый deployment при условии, что этот
набор всё ещё доступен в локальной папке.

## Совместимость

- Плеер без V2 работает через прежний HTTP/ADB.
- V2-плеер без сохранённого токена пульта временно работает как legacy.
- Повторное сопряжение включает защищённый V2.
- Legacy API пока остаётся открытым для миграции старого флота. После
  обновления и повторного сопряжения всех планшетов его необходимо отключить.
