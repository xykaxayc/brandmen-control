# Сервер логов Brandmen Control

Минимальный приёмник: приложение по кнопке «Отправить лог» (вкладка «Логи»)
делает `POST {URL}/logs` с текстом лога; сервер складывает его в файл.

## Контракт

```
POST {URL}/logs?site=<имя ПК>&version=<версия>&ts=<ISO время>
Authorization: Bearer <TOKEN>        # если задан LOG_TOKEN
Content-Type: text/plain; charset=utf-8
<тело: текст лога>
```

Файлы сохраняются в `logs/<site>/<время>.log`.

## Запуск (любой сервер с Node 18+)

```bash
cd server-logs
npm install
LOG_TOKEN=придумай-секрет node server.js   # слушает :8787 по HTTP
```

## HTTPS (обязательно — приложение шлёт только на https)

Сервер отдаёт обычный HTTP; TLS вешается реверс-прокси. Проще всего **Caddy**
(сам получает сертификат Let's Encrypt):

```
# /etc/caddy/Caddyfile
logs.твойдомен.ru {
    reverse_proxy 127.0.0.1:8787
}
```

Нужен домен (А-запись на IP сервера). После этого:

- В приложении → Настройки → «Сервер логов»:
  - Адрес: `https://logs.твойдомен.ru`
  - Ключ: тот же, что в `LOG_TOKEN`

## Где смотреть логи

На сервере в папке `server-logs/logs/<имя-ПК>/`. Каждая отправка — отдельный
файл с меткой времени.

## Альтернативы хостинга

- **VPS** (любой) + Caddy — как выше, полный контроль, дёшево.
- **Render/Railway** — дают HTTPS из коробки, но бесплатный диск эфемерный
  (логи теряются при перезапуске) — подключите том/объектное хранилище.
- **Cloudflare Worker + R2** — без управления сервером; нужно переписать
  приём на Worker (этот пример — Node/Express).

Примечание: на части ПК антивирус перехватывает TLS — приложение для адреса
вашего сервера принимает такой сертификат (как и для GitHub), чтобы отправка
не падала с `CERTIFICATE_VERIFY_FAILED`.
